En el contexto de repositorio multiple para la construccion de un modulo, dividimos la logica por dominio de negocio (area especifica) , 
module "dominio1" { source = repo1 } , de modo que como cada entorno esta aislado es mas fortalace la seguridad. A esto se llega en un momento del
proyecto (en cuestion) , entonces se sigue unos pasos 1)aislar del directorio 2)Historial 3) init y plan 4)Configurar el repo 5)Publicar la version semantica 
6) cambiamos los source al git nuevo.
Si no se hiciera eso todo el proyecto depende del main.tf de la raiz del proyecto , la unica de verdad pero literalmente.
entonces tenemos varios modulos en entornos propios, y pueden versionarse de forma mas limpia, se puede gestionar el versionado semantico
via MAYOR MENOR PATCH , en tanto que la etiqueta firmada es la garantia que esa verion es estable
